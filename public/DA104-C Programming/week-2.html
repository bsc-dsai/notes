Week 2: Lexical Elements in C Programming

In the first Week, we learned how to write a basic C program and the structure of a basic C program. We are now ready for our second Week, which is on lexical elements. What are lexical elements? By lexical elements, we mean the basic building blocks of a C program.

Lexical elements are the basic building blocks of a C program or, for that matter, any programming language. A C program can be thought of as made up of six types of building blocks. These building blocks in C are called tokens. In this Week, we will be understanding all six building blocks. Tokens are the basic building blocks of the language. They are also the smallest units of a C program that have meaning.

Lexical Analysis and Compilation Process

During the compilation process, the compiler breaks the C source code into tokens through a process called lexical analysis. Now, let's look at the compilation process. How does the compiler compile a program?

We have the source code. What is the source code? The source code can be thought of as a sequence of characters, such as int, main, void, etc. These characters are grouped into tokens by the compiler. For example, in a C program:

int main() { 
        // Your code here
    }

This is a sequence of characters. The compiler will group these characters into tokens:

  • int - This is one token (a keyword).
  • main - This is one token (an identifier).
  • () - These are tokens (punctuators).
  • {} - These are tokens (operators).

The compiler groups these characters into tokens, which are the basic vocabulary of the language. It then translates these tokens into object codes. For example:

  • int is a keyword (a token).
  • main is an identifier (a token).
  • () are operators (tokens).
  • {} are punctuators (tokens).

In summary, a C program can be thought of as a sequence of characters, and the compiler groups them into tokens. Each of these tokens is translated into codes.

Types of Tokens in C

In C, there are six types of tokens:

  • Keywords
  • Punctuators
  • Identifiers
  • Constants
  • String Constants
  • Operators

We will be looking into each of these tokens in the upcoming videos.

Week 2: Keywords in C Programming

In this second video, we'll study in detail the first type of token, that is, keywords. What are keywords? Keywords are predefined or reserved words that have special meaning. They are reserved words and they have a special meaning. For example, if I say int, it always means integer. It cannot mean anything else.

In C, there are 32 keywords, which is relatively fewer compared to other languages. That's why we also call C a small language. As we progress in this course, we'll understand each of these keywords and why we call C a small language, because in other languages, the number of keywords is almost double. But here, we only need to learn these 32 keywords.

Let's try to categorize these keywords in different ways. One way is to categorize them depending on whether they're related to data types. Data types refer to the types of identifiers. In this particular slide, I've marked out all the keywords that are related to data types.

Some specific keywords you'll encounter include:

  • auto

  • static

These keywords are encountered when we study functions, and we'll cover them in detail when we look at functions.

Program Example: Calculating Area of a Circle

Now, we have a program here that I'm going to use as a running example to identify all the tokens. This program computes the area of a circle. Let's look at this program in detail.

Comments in C

The first line is a pre-processing directive. The second line is a comment. What are comments? Comments are used for adding notes or explanations within the code. For example, // This program computes the area of a circle helps us understand the purpose of the program.

Comments are ignored by the compiler and do not affect the program. There are two types of comments in C:

  • Multi-line comments - Anything written between /* and */ is a multi-line comment.
  • Single-line comments - Anything written after // is a single-line comment.

For example:

int radius; // This is the radius of the circle

Week 2: Punctuators in C Programming

In this video, we look at the second type of tokens, that is, punctuators. Punctuators in C have a role very similar to that of punctuation in English. Punctuators are used to separate or punctuate the code. What's a punctuator? It has a semantic significance, but does not specify an operation to be performed.

By semantic significance, I mean it has meaning. For example, if I write int radius;, it means this is the end of the statement. It has meaning but it does not perform any operation. That's what I mean by it having significant semantic significance, but not specifying any operation. Punctuators are used to group tokens or terminate a list of tokens, as you can see in this example.

Types of Punctuators in C

  • Braces { } - Braces are used to indicate the start and end of a compound statement. A compound statement consists of multiple statements. For example, in a for loop (we will discuss for loops later), multiple statements can be grouped together using braces. Statements within braces are also known as code blocks.
  • Semicolon ; - The semicolon is used to terminate statements. For example, int radius; terminates the statement.
  • Comma , - The comma can be used both as a separator and as an operator. In this course, we will primarily use the comma as a separator. For example, int radius, area; separates multiple variables in a single declaration.
  • Colon : - The colon is used in conditional statements, like in switch-case statements. It is also used in label statements when using the goto statement. For example, start: indicates a label named "start".
  • Hash # - We've already seen the hash symbol in pre-processing directives. It is used to include libraries, like #include <stdio.h>.
  • Angle Brackets < > - Angle brackets are used in pre-processing directives to include files, as in <stdio.h>.
  • Parentheses ( ) - Parentheses have two main uses: grouping expressions and specifying function parameters. For example, in (x * y) / z, the parentheses group the expression x * y. Parentheses are also used when defining or calling functions, such as in main() or when specifying function parameters.

Example: Identifying Punctuators in a Program

Let's return to our running example program, which computes the area of a circle. Here, we can identify the punctuators:

    return 0;

In this example, the punctuators are:

  • { } - Used to indicate the start and end of a compound statement.
  • ; - Used to terminate statements.
  • , - Used to separate variables.

All the punctuators in this program are marked in green so that you can easily identify them. Punctuators play a key role in structuring C programs.

Week 2: Identifiers in C Programming

In this video, we'll study the third type of token, that is, identifiers. What are identifiers? Identifiers are names that are given to programming elements, particularly used for variables and functions. Identifiers provide unique names to objects in a program. By "objects," we mean variables or function names.

There are two rules to define an identifier:

  • It should be composed of letters, digits, and the special character underscore (_).
  • An identifier must start with either a letter or an underscore. The first character of an identifier should be one of these two.

Examples of Valid Identifiers

Let's look at some examples of valid identifiers:

  • a - This is a valid identifier as it starts with a letter.
  • b - Another valid identifier.
  • principle - A valid identifier that starts with letters.
  • i123 - A mix of letters and digits, which is allowed.
  • i123_xyz - A valid identifier combining letters, digits, and an underscore.
  • _var - Valid as it starts with an underscore.

Examples of Invalid Identifiers

Here are examples of invalid identifiers and why they are not allowed:

  • 1ixyz - Invalid because it starts with a digit. The first character must be a letter or underscore.
  • not#me - Invalid because it contains a special character #, which is not allowed.

An identifier cannot start with anything other than a letter or an underscore, and it cannot contain special characters other than the underscore. It should only include letters, digits, and underscores.

Example: Identifying Identifiers in a Program

Now let's return to our running example of computing the area of a circle. Here, we'll identify the identifiers used in the program:

In this example, the identifiers are:

  • main - This is the name of the function.

All the identifiers are marked in a different color to distinguish them easily. Identifiers are crucial for naming variables and functions in a meaningful way, making the code more readable and easier to document.

Choosing Meaningful Identifiers

When selecting names for identifiers, choose names that are meaningful. For example, if a variable represents the radius of a circle, use radius as the identifier instead of something like ABC or XYZ. Meaningful identifiers enhance the readability and documentation of your program. Use names like radius, area, or circle for clarity and purpose.

Week 2: Constants and String Constants in C Programming

So far, we have seen three types of tokens: keywords, punctuators, and identifiers. In this video, we'll be looking at the fourth one, which is constants, and the fifth one, which is string constants. Constants and string constants are the two types of tokens we'll explore in this video.

Constants

Constants are fixed values that cannot be altered during the execution of a program. There are several types of constants:

  • Integer Constants: These represent whole numbers. Integer constants can be written in different bases:
    • Decimal (Base 10): For example, 0, 77.
    • Octal (Base 8): Octal constants start with a zero, such as 0123. In decimal, this would be calculated as 1(8^2) + 2(8^1) + 3(8^0).
    • Hexadecimal (Base 16): Hexadecimal constants start with 0x. For example, 0xfff.
  • Floating Point Constants: These are real numbers with decimal points. For example, 345.5 is a floating point constant.
  • Character Constants: Individual characters enclosed in single quotes. Examples include 'a', 'b', '\n', and '\t'.

String Constants

String constants consist of more than one character enclosed in a pair of double quotes. For example, "hello" is a string constant. A null string is an empty sequence of characters enclosed in double quotes, like "".

Example: Identifying Constants in a Program

Let's return to our running example of computing the area of a circle and identify the constants:

#include <stdio.h>
// This program computes the area of a circle
int main() {
    float area, radius, pi;
    radius = 5;
    pi = 3.14;
    area = 3.14 * radius * radius;
    printf("\nArea of the circle: %f"\n, area);
    return 0;
}

In this example, the constants are:

  • 5 - This is an integer constant, specifically a decimal constant.
  • 3.14 - This is a floating point constant.
  • "Area of the circle: %f" - This is a string constant, enclosed within double quotes.

Week 2: Operators in C Programming

So far, we have seen five types of tokens: keywords, punctuators, identifiers, constants, and string constants. In this video, we'll start with operators, which are perhaps the most important of all the token types. There are different types of operators, and in subsequent videos, we'll learn how to use them.

What are Operators?

An operator is a token that specifies an operation on at least one operand and yields a result. For example:

c = a + b;

Here, the + is an operator that specifies an operation between two operands, a and b. Operators act on operands to perform calculations or operations. In this case:

  • Operands: a and b are the operands.
  • Operator: + is the operator.

Operands can also be constants. For example, you could write:

c = a + 5;

Here, a is a variable, and 5 is a constant operand. Since the operator acts on two operands, it is called a binary operator.

Unary Operators

An operator that acts on a single operand is called a unary operator. For example:

c = a++;

In this case, the ++ operator acts on a single operand a, making it a unary operator.

Example: Identifying Operators

In this example, the following are operators:

  • () - These parentheses are operators used to call a function.

Week 2: Unary and Arithmetic Operators in C Programming

There are different operators in C. In this video, we'll learn about the unary operators and arithmetic operators. In the last video, we discussed unary operators. A unary operator operates on a single operand. Arithmetic operators, on the other hand, are binary operators, meaning they operate on two operands.

Operators have rules of precedence and associativity:

  • Precedence: The operator with higher precedence is executed first if multiple operators appear together.
  • Associativity: Most operators follow left-to-right associativity, except for unary and prefix operators.

Example: Precedence and Associativity

Let's look at an example to explain precedence and associativity:

a * b / c;

Here, both multiplication and division have the same precedence, but they are evaluated from left to right. So, the result of a * b is computed first, then divided by c. This is an example of associativity.

a + b / c;

In this case, division has higher precedence than addition. Therefore, b / c is computed first, and then the result is added to a. This demonstrates precedence.

Increment and Decrement Operators

Increment (++) and decrement (--) operators can be used as both suffix (postfix) and prefix operators:

  • Postfix: The operation is performed after the statement is executed.
  • Prefix: The operation is performed before the statement is executed.

Postfix Example

c = a++;

If a is 5, then c will get the value 5, and a will be incremented to 6 after the statement is executed.

Prefix Example

c = ++a;

If a is 5, c will get the value 6, as a is incremented before the statement is executed.

Modulo Operator

The modulo operator (%) returns the remainder of a division. For example:

c = 2 % 3;

In this case, 2 % 3 gives a remainder of 2, so c will be 2.

Detailed Examples

Example 1: Integer Division

a * b / c;

If a = 1, b = 2, and c = 3, a * b = 2, and 2 / 3 is an integer division, so the result will be 0.

Example 2: Modulo with Addition

(a * b) % c + 1;

With a = 1, b = 2, and c = 3, first compute a * b = 2, then 2 % 3 = 2, and finally 2 + 1 = 3.

Example 3: Post-increment and Arithmetic

++a * b - c--;

If a = 1, b = 2, and c = 3, the postfix increment will occur after the statement. So, first, compute a * b = 2, and c-- will remain 3 during this statement.

Example 4: Prefix and Subtraction

7 - - b * ++d;

If b = 2 and d = 4, first increment d to 5, then compute 5 * -b (with b as -2), which gives -10. Finally, compute 7 + 10 = 17.

Week 2: Assignment Operator in C Programming

In this video, we'll examine the assignment operator. C treats the = (equal to sign) as an operator.

The syntax is: the variable will be on the left-hand side, and on the right-hand side, we can have a constant or expression.
variable = constant or expression. For example, let's say we have a variable named radius: 

radius = 5;

Here, we have the variable on the left-hand side and a constant on the right-hand side. We can also have an expression on the right-hand side. The associativity of the assignment operator is from right to left. This means that if we have multiple assignment operators in an expression, the evaluation starts from right to left.

Example: Multiple Assignments

Consider this expression:

a = b = c;

Assume the value of b = 3 and c = 5. Since the associativity is from right to left, the expression will first assign c to b. After that, a will get the value of b, which is now 5. At the end of this execution, the result will be:

a = 5, b = 5, c = 5;

Compound Assignment Operators

The assignment operator also has compound forms such as +=, -=, *=, etc. Let’s take an example of +=: 

c += 5;

This is a shorthand for:

c = c + 5;

If the previous value of c was 2, after this statement, the value of c becomes 7.

Example: Multiplication Compound Assignment

Let's take another example with multiplication:

j *= k + 3;

This expands to:

j = j * (k + 3);

If j = 2 and k = 3, the evaluation happens as follows:

  • First, k + 3 is evaluated (since * has higher precedence than +).
  • Then, the result of k + 3 is multiplied by j.

So, j * (3 + 3) = 2 * 6 = 12. Therefore, the final value of j will be 12.

Operator Table and Precedence

We have already seen basic arithmetic operators like +, -, *, /, and % (modulo). In the table of operators, the assignment operator (=) has the lowest precedence of all operators, and its associativity is from right to left.

Week 2: Relational Operator in C Programming

In this video, we will be looking at one more operator, the relational operator. These operators are used to compare values and determine the relationship between them. They are often used when we create conditional statements or expressions. We will learn more about conditional statements in later videos.

The associativity of relational operators is from left to right.

Relational Operators

There are four relational operators:

  • <
  • <=
  • >
  • >=

Let’s take an example:

10 > 7  // True

This expression will evaluate to True.

Another example:

7 >= 7  // True

This will also evaluate to True.

However, an expression like:

7 > 10  // False

Will evaluate to False.

True is usually represented as 1 or any non-zero value, while False is represented as 0.

Operator Precedence

In the operator precedence table, relational operators have a higher precedence than the assignment operator. Their associativity is from left to right.

Relational Expressions

Let’s explore how these relational expressions are evaluated. Consider two variables, a and b, with some examples:

  • If a = 5 and b = 2, then:
    • a < b will evaluate to 0 (False).
    • a > b will evaluate to 1 (True).
    • a <= b will evaluate to 0.
    • a >= b will evaluate to 1.
  • If a = 5 and b = 5, then:
    • a < b will evaluate to 0.
    • a > b will evaluate to 0.
    • a <= b will evaluate to 1.
    • a >= b will evaluate to 1.
  • If a = 2 and b = 5, then:
    • a < b will evaluate to 1.
    • a > b will evaluate to 0.

ASCII Values in Relational Expressions

Sometimes, you'll see expressions with characters in single quotes, like 'a'. These are character constants, which represent ASCII values.

ASCII stands for American Standard Code for Information Interchange. It is a character encoding standard where characters (letters, digits, symbols, or control characters) are represented by numeric values. Computers only understand binary, so each character is assigned a unique 7-bit binary number.

Example: ASCII Values in Expressions

Let’s look at this expression:

'a' + 1 < 'w'

The ASCII value of 'a' is 97, and for 'w', it’s 119. So the expression becomes:

97 + 1 < 119

This evaluates to 98 < 119, which is True (1).

Additional Relational Expression Examples

Let’s evaluate some more complex expressions:

-i - 5 * j >= k + 1
  • If i = 1, j = 2, and k = -7, this expands to:
  • -1 - 5 * 2 >= -7 + 1
  • First, multiplication has higher precedence, so this becomes:
  • -1 - 10 >= -6
  • Now, the subtraction:
  • -11 >= -6, which is False (0).

Another example:

3 < j < 5
  • If j = 2, the expression becomes:
  • 3 < 2 < 5
  • Since 3 < 2 is False (0), this simplifies to:
  • 0 < 5, which is True (1).

Unexpected Results

Lastly, let’s look at this expression:

x = 7e + 33, y = 0.001

x < x + y

Where x is a very large number and y is small. Due to the limitations in the machine's representation of numbers, y may be ignored. This causes the expression to evaluate as:

x < x

Which is False (0), resulting in an unexpected outcome.

Week 2: Equality and Logical Operators

In this video, we'll look into two operators: the equality operator and the logical operator. The equality and the logical operators are used to compare values and make decisions based on the comparison result.

Equality Operators

We have two equality operators:

  • Equal to (==): Checks if the two values are equal. For example, if 7 == 7, this evaluates to true. It checks if the expression on the left-hand side is equal to the right-hand side, and if that is true, then it will evaluate to true.
  • Not equal to (!=): Checks if the two values are not equal. For example, if 7 != 6, this is true because 7 is not equal to 6. This will evaluate to true.

These are binary operators and yield either true or false. They take two operands.

Example

Consider the expression i + j + k == -2 * -k with i = 1, j = 2, and k = 3. Let's evaluate:

  • Left-hand side: i + j + k = 1 + 2 + 3 = 6
  • Right-hand side: -2 * -k = -2 * -3 = 6

The expression evaluates to 6 == 6, which is true.

Logical Operators

There are three logical operators:

  • Logical NOT (!): A unary operator that works on a single operand. If the operand is zero, !a evaluates to true (1). If the operand is non-zero, !a evaluates to false (0).
  • Logical AND (&&): A binary operator that returns true if both conditions are true. It has a higher precedence than logical OR.
  • Logical OR (||): A binary operator that returns true if at least one of the conditions is true. It has a lower precedence than logical AND.

Operator Precedence and Associativity

Operator Precedence Associativity
Logical NOT (!) Highest (same as unary operators) Right to Left
Logical AND (&&) Higher Left to Right
Logical OR (||) Lower Left to Right
Equality (==, !=) Lower than relational operators Left to Right

Examples

Let's look at some examples to understand how these operators work:

  1. Expression: i && j && k with i = 3, j = 3, and k = 3. This evaluates to true because all operands are non-zero.
  2. Expression: x || (i && (j - 3)) with x = 0.0, i = 3, j = 3. This evaluates to false because 3 - 3 is zero and 0 || 0 is false.
  3. Expression: i < j || x < y with i = 3, j = 3, x = 0.0, and y = 2.3. This evaluates to true because 3 < 3 is false but 0.0 < 2.3 is true.
  4. Expression: (c - 1) || (c + 1) with c = 66. This evaluates to true because 66 - 1 is true, so the overall expression evaluates to true without needing to evaluate the second part.

In this video, we have covered two types of operators: equality operators and logical operators. We have seen examples of how these operators are evaluated and how their precedence and associativity affect the result.
