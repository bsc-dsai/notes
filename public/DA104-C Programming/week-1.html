Week 1: Computer System Fundamentals Part 1

We have had an overview of how the course will run in the last few videos. Today we are ready to start with the actual learning. We'll begin with some computer system fundamentals. We'll have an overview of what a computer system is, including its hardware and software components. However, learning the entire computer system is a huge task, so we'll focus on the computer system from the perspective of programming.

Overview of Computer System Components

A computer system, as you can see, consists of four main components. At the top, we have the users— that's you and me. The users generally interact with an application software, but they can also interact directly with the operating system. The application software runs on top of the operating system, and the operating system controls the hardware. This is how the entire system works together.

Examples of Application Software

Some examples of application software include Microsoft Word. In this course, we'll be using a compiler extensively, and a compiler is also an example of application software.

Examples of Operating Systems

Examples of operating systems include Microsoft Windows, Linux, Mac OS, and so on. Finally, we have hardware. The hardware we are considering here includes your laptop or desktop.

Hardware: System Unit

Let's look in more detail at the hardware. The hardware components of a computer system include your monitor, keyboard, and speakers, but our primary focus is the system unit. While it's often called the CPU, its actual name is the system unit.

Power Supply (SMPS)

One of the first things we'll see inside the system unit is the power supply, also called the SMPS (Switched Mode Power Supply). Its function is to convert AC (Alternating Current) to DC (Direct Current). All components within the computer system operate on DC power.

Motherboard and CPU

On the motherboard, the central component we'll focus on is the CPU (Central Processing Unit). The CPU performs all the processing, including arithmetic and logic operations, and controls the hardware. The instructions for our programs (like C programs) are stored in the memory, which is the RAM (Random Access Memory).

Memory and Storage

The RAM is volatile, meaning once you write your program, you need to save it. When saved, it is stored in the secondary storage, which can be a hard disk drive (HDD) or a solid-state drive (SSD).

The von Neumann Architecture

John von Neumann, a famous scientist, proposed the von Neumann architecture, which simplifies the study of computer systems. Most modern computer systems follow this architecture. The von Neumann architecture consists of four components:

  • Memory: Stores data and programs. It is volatile and requires secondary storage to save data.
  • Control Unit: Executes programs and manages the flow of data.
  • Arithmetic and Logic Unit (ALU): Performs arithmetic and logic operations requested by the program.
  • Input and Output (I/O): Devices that allow interaction with the computer system.

All these components are connected by a bus, a communication subsystem that links the components of the system unit.

Memory Structure

Let's look more closely at the memory. There are two important aspects:

  • Storage Unit: This is where the actual bits and bytes are stored.
  • Address: This refers to the specific location in memory where data is stored.

The size of the memory determines how much data can be stored. For example, if N = 4, then you can have 2^4 storage units. Another important factor is the width of the memory, which refers to the number of bits that can be read at one time. A 32-bit CPU can read 32 bits at a time, and a 64-bit CPU can read 64 bits at a time.

Memory Sizes

The typical memory sizes for personal computers and laptops are measured in gigabytes. Other memory sizes include kilobytes, megabytes, and terabytes.

Week 1: Computer System Fundamentals - Part 2

Welcome to the second part of the computer system fundamentals. In the last video lecture, we discussed hardware and its various components. Today, we'll focus on the higher-level components: the operating system and application software.

Overview of Application Software

This is an overview of the application software in a computer system. As seen in the slide, there are multiple users, each performing different tasks. For example:

  • User 1: Working on a compiler
  • User 2: Using an assembler
  • User 3: Editing text in a text editor
  • nth User: Managing a database system

All these application software programs run on top of the operating system.

What is an Operating System?

An operating system (OS) acts as an interface between application software and hardware. It serves as a bridge between the two.

Formal Definition: An operating system is a layer between the application software and the hardware.

Goals of an Operating System

  • Abstraction: It hides the hardware details. For example, when you save a program, the OS takes care of saving it to the hard disk, while hiding the underlying hardware processes from the user.
  • Efficient Use of Hardware: The OS allows multiple users to share the same hardware, making it seem as if each user has full control over the system.
  • Protection: The OS ensures that users cannot see or access each other's work unless permitted.

What is Application Software?

While the OS is called system software because it controls the hardware, application software is designed to perform specific tasks. Some examples include:

  • Compiler: Translates a C program into machine language.
  • Text Editor: Used for typing text.
  • Database System: Used for saving records.

Machine Code and High-Level Languages

The computer understands only binary code (zeros and ones), known as machine code. While machine code is very efficient for the computer to decode, it's difficult for humans to read and write. That's why we have high-level languages such as:

  • C
  • Fortran
  • Java
  • Python

High-level languages use English-like syntax, making them easier for humans to understand.

The Role of a Compiler

Since computers only understand machine code, we need a compiler to translate high-level language instructions into machine code.

Week 1: Introduction to Algorithms

In this video, we will learn about an important concept: algorithms.

What is an Algorithm?

Flowchart is nothing but a graphical representation of an algorithm. They mean the same thing. In this video, I'll give you an introduction to algorithms and their definition. Later on in your program, you will have a full course on algorithms.

A typical programming task involves two phases:

  1. Problem-Solving Phase: In this phase, we produce an ordered sequence of steps that describes a solution to the problem. Emphasis on the word "ordered" means that steps must be followed in a specific sequence. For example, to make French fries, you first cut the potatoes and then put them in the frying pan. You cannot do it in reverse. This sequence of steps is called an algorithm.
  2. Implementation Phase: In this phase, we translate the sequence of steps into a program using a programming language. In this course, we will use the C programming language to translate the sequence of steps into a program.

For very simple problems, you might not need to write a detailed algorithm and can start programming directly.

Example: Finding the Largest of Two Values

Let's understand algorithms with an example. The task is to read in two values and find out which one is larger. We want to print this with an identifying message, such as "The largest of the two is 7." Here is the algorithm for this task:

  1. Read in two values: value 1 and value 2.
  2. Compare the two values:
    • If value 1 is greater than value 2, assign value 1 to max.
    • If value 1 is not greater than value 2, assign value 2 to max.
  3. Print "The largest value is max."

Characteristics of an Algorithm

  • Takes Input: An algorithm takes input values.
  • Produces Output: An algorithm produces output values.
  • Simple and Unambiguous Steps: The steps are clear and straightforward. For example, comparing value 1 with value 2 and assigning the larger value to max.
  • Finite Time: Each step of the algorithm takes a finite amount of time to complete. For example, assigning a value to max happens in a finite time.

Converting Algorithms to Flowcharts

Now, let's convert the above algorithm into a flowchart. In a flowchart:

  • Input and Output: Represented by parallelograms. For instance, taking input values value 1 and value 2 is shown with a parallelogram.
  • Comparison: Represented by diamond shapes. Comparing value 1 with value 2 is shown with a diamond shape.
  • Processing: Represented by rectangles. Assigning value 1 or value 2 to max is shown with a rectangle.
  • Flow of Control: Shown by arrows connecting the shapes, indicating the sequence of steps.

What is Linux?

Linux is a Unix clone written from scratch by Linus Torvalds, with assistance from a loosely-knit team of developers across the net. By "loosely-knit," it means they were not paid but collaborated out of interest. It is a clone of Unix, meaning its operation is very similar to Unix.

What is Unix?

Unix is a multitasking operating system, meaning a user can perform multiple tasks simultaneously. For example, you can compile code, write a C program, and listen to music at the same time. Unix is also multi-user, allowing multiple users to work on the same machine concurrently. While Windows is popular among desktop operating systems, Linux is prevalent among web servers due to its resistance to viruses.

Linux Distributions

When a user wants to obtain the Linux operating system, they typically download one of the Linux distributions. A Linux distribution includes the Linux kernel and additional software libraries and utilities from the GNU project, creating a complete operating system. Some popular distributions are:

  • Ubuntu: A user-friendly distribution known for its ease of use.
  • Debian: Known for its stability and large repository of software.
  • CentOS: A distribution designed for enterprise environments, based on Red Hat Enterprise Linux.

Connecting to a Linux System

To connect to a Linux system, you can:

  1. Log in directly: Sit in front of a terminal and log in using a username and password, similar to Windows.
  2. Connect remotely: You can use text mode through SSH (Secure Shell) or a graphical interface via RDP (Remote Desktop Protocol) from Windows, providing a Windows-like environment on Linux. Note that graphical interfaces may require more bandwidth compared to text mode.

The Shell

The shell is a command interpreter that processes commands entered in the terminal. It communicates with the core of the operating system (the kernel). Popular shells include:

  • bash (Bourne Again Shell): The default shell in many Linux distributions. For this tutorial, we’ll use bash. Shell commands are case-sensitive, so LS and ls are different.

Getting Help in Linux

If you need help with a command, you can use:

Week 1: History of C

C has been designed by Dennis Ritchie and evolved from two previous languages, BCPL and B.

BCPL stands for Basic Combined Programming Language, and B was its simplified successor.

Since C evolved from B, it was named as C. Dennis Ritchie developed C to create the UNIX operating system, and for this contribution, he received the Turing Award, which is the highest honor in computer science.

C was designed in the early 1970s, around 1972-73. By the late 1970s, it had evolved into what is called Traditional C, with multiple flavors available. In 1989, the American National Standard Institute (ANSI) standardized C, creating ANSI C, which was later updated in 1999.

Week 1: Hello World Program in C

In this video, we typed out the "Hello World" program in a text editor, compiled it, and ran it. When we executed the program, it produced the output: Hello World. Let's now take a closer look at what the program does.

Preprocessing Directive

The first line of the program starts with a #. Lines that begin with this symbol are called preprocessing directives. These lines are processed by the preprocessor, which is not part of the compiler. The preprocessor's job is to include the contents of the file stdio.h in the program. This file contains information required for functions like printf, which we use to display text.

The angle brackets around stdio.h (i.e., <>) indicate that the file is located in the standard header file directory. In Linux, for example, it is typically found in /usr/include.

Main Function

main is the name of a function. It is the first function that the compiler will execute. The int before main tells the compiler that this function will return an integer. The keyword void inside the parentheses means that the function takes no parameters.

The braces ({}) denote the beginning and end of the function, which is referred to as the function body. These braces are also called punctuators in C.

Printf Function

The printf function is responsible for printing text to the console. The string "Hello World" is passed as an argument to printf, which then displays it. The escape sequence \n represents a newline character, so after printing "Hello World," the cursor moves to the next line.

Return Statement

Finally, we have return 0;. This statement tells the program to return the value 0 to the operating system, indicating that the program has executed successfully. The reason we use return 0 is that we declared the main function to return an integer (int), and this is fulfilling that requirement.

Semicolons in C

All C statements, such as printf and return, must end with a semicolon (;). This is a fundamental syntax rule in C.

Week 1: Converting Algorithm to C Code: Largest of Two Values

In this section, we will complete the task of converting an algorithm to a C program. This process involves two main phases:

  1. Problem-solving phase: Writing an algorithm that describes a solution to the problem.
  2. Translation phase: Translating the algorithm into a C code.

We'll convert the following algorithm, which finds the largest of two values, into a C program:


Algorithm:
1. Start
2. Read value1, value2
3. If value1 > value2, then
   max = value1
   else max = value2
4. Print max
5. Stop

Step-by-Step C Code

Let’s walk through the conversion of the algorithm into a C code.

Including Libraries

The first step is to include the necessary libraries for input and output. For the printf and scanf functions, we use the #include <stdio.h> directive: 

#include <stdio.h>

Writing the Main Function

Next, we define the main function, which is the entry point of a C program. It returns an integer type (int) and does not take any parameters, so we use void:

int main(void) {

Declaring Variables

We need to declare the variables that will store the two input values and the maximum value. We declare them as integers:

int val1, val2, max;

Reading Input

To read the input values, we use the scanf function. Since we are reading integers, we use the format specifier %d along with the addresses of the variables (using &):

scanf("%d %d", &val1, &val2);

Comparing Values

Next, we compare the two values using an if-else statement. If val1 is greater than val2, we assign val1 to max. Otherwise, we assign val2 to max: 

if (val1 > val2) {
        max = val1;
    } else {
        max = val2;
    }

Printing the Result

Finally, we print the maximum value using the printf function:

printf("The largest value is: %d", max);

Ending the Program

We close the main function with a return statement to indicate successful execution:

return 0;
}

Complete C Program

Here is the full C program to find the largest of two values:

#include <stdio.h>

int main(void) {
    int val1, val2, max;
    scanf("%d %d", &val1, &val2);

    if (val1 > val2) {
        max = val1;
    } else {
        max = val2;
    }

    printf("The largest value is: %d", max);
    return 0;
}

You can now run this program in your lab and check the output by providing different input values.

Week 1: Input and Output in C

In this section, we'll explore input and output statements in C, which allow users to interact with a program. Two of the most commonly used statements are scanf for input and printf for output. You've already seen printf in action in the "Hello World" program. Let's now dive deeper into these functions.

Using scanf for Input

The scanf function is used to read input from the standard input (usually the keyboard). The function is declared in the stdio.h library, so we need to include this file when using input and output statements in our program.

scanf works by reading formatted input, which requires format specifiers to indicate the data type being read. For example, %d is used to read an integer. Here is how it works:


    int radius;
    scanf("%d", &radius);  // Read an integer and store it in 'radius'

In this case, %d is the format specifier for an integer, and &radius refers to the address of the variable where the input will be stored. Other format specifiers include:

  • %c: For reading a character
  • %f: For reading a floating-point number
  • %s: For reading a string

Using printf for Output

The printf function outputs data to the standard output (usually the monitor). Like scanf, printf uses format specifiers to format the output. For example, to print an integer, use %d. 


    printf("The value of radius is %d", radius);  // Output the value of 'radius'

In this case, %d will be replaced by the value stored in radius. Here are some other format specifiers:

  • %c: For printing a character
  • %f: For printing a floating-point number
  • %s: For printing a string

Example Program: Adding Two Integers

Let's write a program that uses scanf to input two integers and printf to output their sum:


    #include <stdio.h>

    int main() {
        int num1, num2, sum;

        // Input two integers
        printf("Enter two integers: ");
        scanf("%d %d", &num1, &num2);

        // Compute the sum
        sum = num1 + num2;

        // Output the result
        printf("The sum of %d and %d is %d\n", num1, num2, sum);

        return 0;
    }

In this program, scanf reads two integers and stores them in num1 and num2. The sum is then computed and stored in sum. Finally, printf outputs the result using format specifiers %d to print the values of num1, num2, and sum.

Conclusion

In this section, we've learned how to use the scanf and printf functions for formatted input and output in C. These two functions are essential for reading user input and displaying output, and you'll continue to use them as you explore more C programs.
